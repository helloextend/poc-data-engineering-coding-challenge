# Interviewer docs

Internal-only docs for running the 60-minute Principal Data Engineer live interview. Not shared with candidates. Start here if it's your first time running this interview.

## Onboarding path

Read in this order:

1. [`technical-interview-design.md`](./technical-interview-design.md) — design doc: why this interview exists and what it tests.
2. [`stakeholder-brief-jamie.md`](./stakeholder-brief-jamie.md) — Jamie's posture and what to avoid.
3. [`interviewer.md`](./interviewer.md) — the run sheet. Skim live during the call.
4. [`pr-grading-rubric.md`](./pr-grading-rubric.md) — how the PR is graded after the call.

## Usage matrix

| File | Audience | When | How to use |
|---|---|---|---|
| `interviewer.md` | Interviewer | DURING the call | Skim live; quote verbatim scripts; cross-check planted cases |
| `pr-grading-rubric.md` | Grading agent (+ interviewer for Section B notes) | AFTER the call | Interviewer fills Section B from notes; agent grades from PR + rubric |
| `answer-key.md` | Interviewer + grading agent | DURING (cross-check) + AFTER (rubric ground truth) | Lookup; regenerated by `setup/generate.py` |
| `stakeholder-brief-jamie.md` | Interviewer | ONBOARDING + 5 min pre-call | Internalize Jamie's posture |
| `technical-interview-design.md` | Interviewer | ONBOARDING only | Read once when first learning to run this interview |
| `_answer_key.py` | `setup/generate.py` (programmatic) | Triggered by `make seed` only | Regenerates `answer-key.md` from a fresh seed run. Not triggered by `make setup`. |
| `MAINTAINERS.md` | Engineers maintaining the rig | When regenerating seeds or modifying the planted bug | Build modes, seed regeneration checklist, pre-handoff scrub. |

Operator's note on `technical-interview-design.md`: read it once for *why*; the live operational content is all in `interviewer.md`. Don't over-internalize — too much knowledge of the trap risks leading the candidate.
